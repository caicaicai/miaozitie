# 🐱 喵字帖 - 小学生练字工具

一个专为小学生设计的在线汉字练字帖生成工具，支持笔画演示、描红练习和文章注音。

## 🌐 在线体验

[喵字帖](https://miaozitie.80wdb.com/) - Cloudflare Pages 部署版本

## ✨ 功能特点

### 🏠 导航首页 (`index.html`)
- 🗂️ **工具卡片导航**：一进站即可看到全部 6 个工具的卡片入口，点击即可进入
- 🎨 **专业简洁设计**：独立于各功能页面的品牌横幅 + 卡片网格布局

### 🎯 互动自测练字 (`zice.html`)
- ✍️ **在线书写判定**：基于 Hanzi Writer 笔顺引擎，鼠标/触屏按笔顺书写，实时判断对错
- 💡 **提示笔顺**：卡住时可查看完整笔顺动画演示，再继续书写
- 🎚️ **难度可调**：可开关提示轮廓、设置连续写错几次后自动提示
- 📊 **测验结果统计**：完成后按"完美/良好/需多练"分类展示，可只重测写错的字

### 📝 笔画练习模式 (`bihua.html`)
- 📚 **年级分类**：支持一到六年级（上下册）的生字库，共2500+个常用汉字
- 🖊️ **笔画演示**：逐笔显示汉字书写顺序，帮助学习正确笔顺
- ✍️ **描红练习**：提供田字格描红格和空白练习格，数量可分别设置
- 🔤 **拼音标注**：自动显示汉字拼音，学字认音同步进行
- 🎨 **自定义设置**：可调节格子大小、文字透明度、练习次数等
- 🖨️ **打印优化**：支持单列/双列布局，适配A4纸张

### ✍️ 抄字模式 (`chaozi.html`)
- 📐 **紧凑布局**：每个字占用极小空间，适合抄写文章
- 🔧 **灵活配置**：
  - 显示/隐藏拼音、原字、笔顺
  - 笔顺列数（3-12列）
  - 笔顺行数（1-6行）
  - 描红/空白练习格
- 📊 **多列打印**：支持 6列 / 8列 / 10列 布局
- 📏 **统一高度**：所有字的容器高度一致，布局整齐美观
- ➖ **分割线**：行列分割线，视觉更清晰

### 📖 文章注音 (`pinyin.html`)
- 📝 **批量转换**：输入任意中文文章，自动添加拼音注音
- 🎨 **样式定制**：可调节字体大小、拼音颜色、是否显示声调
- 📚 **预设故事**：内置多个经典故事示例
- 🖨️ **打印友好**：优化的打印样式，适合儿童阅读

### 🔊 生字听写 (`tingxie.html`)
- 🎲 **年级抽取 / 自定义内容**：可按年级随机抽取生字，也可自定义听写词语列表
- 🔊 **语音朗读**：调用浏览器语音朗读生字/词语，支持语速、朗读遍数设置
- ⏯️ **自动 / 手动播放**：自动定时播放下一个，或手动点击切换，适合不同节奏
- 📋 **答案核对 + 打印答题纸**：听写完成后可查看拼音答案，或打印田字格答题纸

### 🔤 看拼音写汉字 (`moxie.html`)
- 🎲 **年级抽取 / 自定义内容**：与生字听写一致的内容来源方式
- 📝 **默写练习纸**：每个字/词上方标注拼音，下方留出多个田字格供反复默写
- 🔁 **练习次数可调**：每个内容可设置 2~6 次重复练习
- 🖨️ **打印友好**：适配 A4 纸张，可直接打印使用

### 🏷️ 姓名贴生成器 (`mingtie.html`)
- ✏️ **一键批量**：输入班级、姓名、学号，自动生成整页姓名贴
- 🔢 **数量可调**：自定义打印数量，自动铺满 A4 页面
- 🖨️ **打印优化**：适配 A4 纸张，方便裁剪粘贴

### 🧮 口算速练 (`kousuan.html`)
- ➕ **多种运算**：支持加减乘除及混合运算，可自定义数值范围
- ⌨️ **数字键盘**：内置虚拟数字键盘，手机 / 平板均可流畅答题
- 🎯 **即时反馈**：自动判分、记录正确率，越算越熟练
- 🌙 **护眼模式 / 全屏模式**：适合长时间练习场景

### 📐 运算表打印 (`yunsuanbiao.html`)
- ✖️➕➖ **三种运算表**：乘法表（经典九九表样式）、加法表、减法表
- 🔢 **范围可调**：乘法表支持 1~6/9/10/12，加减法表支持 5~50 以内
- 🕳️ **填空练习模式**：可随机挖空部分格子，打印后当堂练习
- 🖨️ **一键打印**：适配 A4 纸张

### 🕐 时钟认读练习 (`shizhong.html`)
- 🎮 **互动答题**：随机显示模拟时钟，选择时/分作答，即时判断对错并统计正确率
- 🖨️ **打印练习卷**：批量生成多个时钟图案 + 空白作答线，适合课堂练习
- 🎯 **精度可调**：整时 / 5 分钟 / 任意分钟三档难度

### 📜 古诗词打印卡 (`gushi.html`)
- 📚 **课本必背古诗**：内置 23 首常见小学必背古诗
- 🔤 **自动拼音注音**：基于 Pinyin Pro 自动为每个字标注拼音
- 📖✍️ **两种卡片模式**：背诵卡（显示全文+拼音）/ 默写卡（只留空行）
- 🎲 **随机抽选**：可一键随机选取 5 首生成打印

### 🗓️ 课程表生成器 (`kechengbiao.html`)
- 🔧 **节数与星期范围可调**：5~10 节、周一到周五/六/日均可
- ✏️ **直接填写**：生成的表格每格都是可编辑输入框，填好课程名称即可打印
- 🎨 **多种主题色**：清新蓝/活力粉/自然绿/阳光橙

### 🔍 形近字辨析练习 (`xingjinzi.html`)
- 📝 **四选一辨析**：句子留空，四个形近字中选出唯一正确答案（如 己/已/巳、做/作/坐/座）
- 💡 **记忆小贴士**：答题后展示该组形近字的结构和用法区别
- 📚 **内置 66 题**：覆盖约 30 组小学常见易混淆字
- 📊 **错题回顾**：练习结束后列出答错的题目方便复习

### 💰 人民币凑钱练习 (`renminbi.html`)
- 🪙 **点击面额凑钱**：随机给出目标金额，点击人民币面额凑出相同金额
- 🎯 **难度可调**：仅整元（简单）/ 元+角（较难）
- 📊 **正确率统计**：完成一组题目后显示得分

### 🔢 报数计时 (`baoshu.html`)
- 🔢 **正计时 / 倒计时**：设置一个数字，选择从 1 数到该数字，或从该数字倒数到 1
- ⏱️ **快捷计时按钮**：1/3/5/10 分钟一键开始倒计时，自动按"分:秒"格式显示（如 03:00）
- ⏱️ **间隔可调**：每个数字之间的停留时间可选 0.5～3 秒
- 📢 **报时间隔可设置**：默认每个数字都报，也可以改成每 5/10/30/60 个才报一次，适合数字很大或长时间倒计时的场景（不再是自动隐藏逻辑，完全由用户设置控制）
- 🔊 **语音报数 / 提示音 / 静音**：可以让浏览器读出每个数字，也可以用哔声代替，或者只看画面不出声；分:秒模式下遇到整分钟会读成"X分钟"更自然
- 🖥️ **专门的计时画面**：不管是手动开始还是点快捷按钮开始，都会自动切换到全屏式的计时界面（隐藏设置区），并有"⏱️ 计时进行中"的醒目提示，避免看不到计时状态
- 🎬 **动画 + 进度**：数字出现时有弹出动画，倒计时结束会有"时间到"提示，配合进度条显示第几个

### 🧪 AI 内容小助手 (`ai-zhushou.html`) · 实验性功能
- 🤖 **浏览器本地大模型**：基于 [Transformers.js](https://github.com/huggingface/transformers.js) + [MiniCPM5-1B](https://github.com/OpenBMB/MiniCPM)，模型完全在用户浏览器本地运行，不经过任何服务器
- 💬 **对话调试窗口**：这个页面本身只负责"加载模型 + 简单对话调试"，用来测试模型效果
- 🔌 **各功能页面自动感知**：模型加载逻辑抽取到共享文件 `ai-engine.js`；只要在任意页面成功加载过一次模型，`tingxie.html`（生字听写）、`moxie.html`（看拼音写汉字）等页面会自动感知到，并出现"🤖 AI 辅助生成词语"入口；没加载过则显示引导链接
- 🗑️ **一键清理模型缓存**：可以清空浏览器里已下载的模型文件（释放存储空间，或用来强制重新下载排查问题），清理后其它页面的"AI 辅助生成"入口会自动恢复为"未加载"状态
- ⚡ **WebGPU 加速，自动降级**：支持 WebGPU 的设备会更快，不支持则自动退回较慢的 CPU(WASM) 模式
- ⚠️ **重要说明**：
  - 依赖社区提供的浏览器版 ONNX 模型文件（非官方），可能不稳定
  - 首次使用需下载约 500MB+ 模型数据（仅一次，浏览器会缓存），模型直接从 Hugging Face Hub 下载，不占用本项目仓库空间，也不受 Cloudflare Pages 单文件 25MB 限制的影响
  - 模型用的是 4bit 量化格式，**主要依赖 WebGPU 显卡加速**：完全没有可用显卡时可能直接加载失败而不只是变慢；即使检测到 WebGPU，**集成显卡（核显）对 1B 参数模型来说仍然会比独立显卡慢很多**，这是硬件性能差异，不是 bug
  - 生成内容**仅为草稿**，请老师/家长审核后再使用，不要直接给孩子使用未经检查的内容
  - 模型权重遵循 OpenBMB「[通用模型许可协议](https://github.com/OpenBMB/General-Model-License)」：学术研究免费，商业用途需邮件登记（cpm@modelbest.cn）后免费授权，请根据实际部署场景自行确认是否需要登记

### 使用步骤

打开 `index.html` 首页，点击工具卡片即可进入对应功能；每个功能页顶部都有"← 返回首页"链接，方便随时切换。

#### 笔画练习
1. 选择年级（可选）
2. 输入汉字或点击"随机生成"
3. 调整描格/练格数量、透明度等参数
4. 点击"打印练字帖"

#### 互动自测练字
1. 按年级随机抽取，或输入自定义汉字
2. 设置提示轮廓和自动提示时机
3. 点击"开始自测"，按笔顺在方格中书写
4. 完成后查看测验结果，可只重测写错的字

#### 抄字模式
1. 输入要抄写的文章或句子
2. 配置显示选项（拼音/原字/笔顺）
3. 选择练习格类型（描红/空白）
4. 调整打印列数
5. 点击"打印"

#### 文章注音
1. 输入或粘贴中文文章
2. 调整字体大小和拼音颜色
3. 点击"生成注音"
4. 点击"打印"

#### 生字听写
1. 按年级随机抽取，或输入自定义听写内容
2. 设置语速、朗读遍数、自动/手动播放方式
3. 点击"开始听写"，孩子边听边写
4. 听写完成后查看答案核对，或打印听写答题纸

#### 看拼音写汉字
1. 按年级随机抽取，或输入自定义字词
2. 设置每个内容的练习次数
3. 点击"生成练习纸"并打印

#### 姓名贴生成器
1. 输入班级、姓名、学号
2. 设置打印数量
3. 点击"生成姓名贴"并打印

#### 口算速练
1. 选择运算类型和数值范围
2. 点击开始练习，用虚拟数字键盘答题
3. 查看正确率统计，可开启护眼/全屏模式

#### 运算表打印
1. 选择乘法表/加法表/减法表及范围
2. 可选开启填空练习模式
3. 点击"生成运算表"并打印

#### 时钟认读练习
1. 互动练习：设置时间精度和题量，点击"开始练习"，选择时/分作答
2. 打印练习卷：设置数量和精度，点击"生成练习卷"并打印

#### 古诗词打印卡
1. 勾选要练习的古诗（或随机选 5 首）
2. 选择"背诵卡"或"默写卡"模式
3. 点击"生成卡片"并打印

#### 课程表生成器
1. 设置节数、星期范围和主题色
2. 点击"生成课程表"，在表格中直接填写课程名称
3. 点击打印

#### 人民币凑钱练习
1. 选择难度（整元/元+角）和题目数量
2. 点击"开始练习"，点击人民币面额凑出目标金额，凑够自动判对进入下一题

#### 报数计时
1. 选择"正计时"或"倒计时"，设置要数到的数字；或直接点击"1/3/5/10 分钟"快捷按钮一键开始倒计时
2. 设置间隔时间、报时间隔（多久报一次）和报数方式（语音/提示音/静音）
3. 点击"开始报数"，画面会切换到专门的计时界面，可随时暂停/继续或结束

#### 形近字辨析练习
1. 选择题目数量
2. 点击"开始练习"，从四个字中选出能让句子正确的那个
3. 查看记忆小贴士，练习结束后复习答错的题目

#### AI 内容小助手（实验性）
1. 打开 `ai-zhushou.html`，阅读页面顶部的使用须知（模型体积、设备要求、内容需人工审核等）
2. 点击"加载模型"，等待下载完成（首次使用耗时较长），加载完成后可以在对话窗口里随便测试模型效果
3. 加载完成后，打开"生字听写"或"看拼音写汉字"页面，会自动出现"🤖 AI 辅助生成词语"入口
4. 填写年级/数量/主题后点击生成，复制生成内容到文本框里，人工检查无误后再使用

### v2.9.4 (2026-07)
- 🐛 修复语音报数排队积压导致越报越滞后的问题（比如画面数到140、语音还停留在156）：改成"如果上一个数字还没读完就跳过这一次报数"，不打断、也不排队，画面永远准时，只是语速跟不上时会少报一些数字。用 Node 脚本模拟验证了排队策略会无限积压延迟，跳过策略延迟恒为 0
- 💡 补充使用说明：如果语音经常被跳过，建议调大"每个数字间隔"或"报时间隔"

### v2.9.3 (2026-07)
- 🐛 修复报数计时"开始后画面是空的，只看到标题"的问题：计时舞台容器少加了一个用来控制显示的 CSS 类，导致内容永远不可见，现在改成容器始终可见、由内部子元素各自控制显示
- 🐛 修复语音报数在数字较大（如超过100）时听起来"两个数重叠/含糊不清"的问题：原因是每次报数前都会打断上一个还没读完的数字，改成不打断、让语音按顺序排队读完，不再重叠（此版本随后发现会积压延迟，已在上面这条继续修复）

### v2.9.2 (2026-07)
- 🐛 修复报数计时体验问题：开始计时（无论手动还是快捷按钮）后画面会自动切换到专门的计时界面（隐藏设置区），并有"⏱️ 计时进行中"提示，避免用户看不到当前在计时
- ✅ "报数间隔"改为显式设置项（默认每个都报，可选每5/10/30/60个报一次），移除了之前自动隐藏触发的节流逻辑，行为更可预期，手动开始和快捷开始遵循同一套设置

- ✅ 首页新增对应卡片入口

### v2.8.6 (2026-07)
- 🐛 针对手机端测试反馈的两个问题做了改进：
  - Safari 仍可能卡在"正在检测设备加速能力..."：加了第二层独立的看门狗超时（6 秒），双重保险
  - Chrome 下载完模型后页面自动刷新、看不到对话框、重新点加载又要重新下载：这通常是手机内存不足导致页面被系统重新载入。现在会在页面加载时检测"上次加载是否被异常打断"，并给出明确提示，同时加了 `overscroll-behavior` 防止下拉刷新手势意外触发
- ⚠️ 新增手机设备检测提示：在使用须知里明确告知手机上这个功能成功率较低、更推荐用电脑浏览器

### v2.8.5 (2026-07)
- 🐛 修复手机浏览器上页面卡在"正在检测设备加速能力..."不动的问题：部分手机浏览器的 `navigator.gpu.requestAdapter()` 会一直不返回结果，`detectGPUAdapter()` 现在带了超时保护（最多等 4 秒），超时后会提示"检测超时，仍可尝试加载模型"，不会再卡死页面

### v2.8.4 (2026-07)
- ✅ 新增"🗑️ 清理模型缓存"功能（`ai-zhushou.html`）：优先用 Transformers.js 自带的 `ModelRegistry.clear_pipeline_cache()` 清理，失败则回退到直接操作浏览器 Cache Storage
- ✅ 清理后会自动清除"模型已加载过"的标记，其它页面（生字听写、看拼音写汉字）的 AI 辅助生成入口会恢复为"需要先加载模型"的引导状态

### v2.8.3 (2026-07)
- ✅ 新增 `detectGPUAdapter()` 显卡检测：实际调用 `navigator.gpu.requestAdapter()` 并读取适配器信息，区分"完全没有 WebGPU""拿到的是软件模拟适配器""检测到真实显卡"三种情况，不再只是简单判断 API 是否存在
- ✅ `ai-zhushou.html` 的设备状态提示改为展示检测到的显卡信息，并明确提示：q4 量化模型主要依赖 WebGPU（无显卡可能直接跑不起来，不只是慢），集成显卡（核显）跑 1B 模型仍会比独立显卡慢很多

### v2.8.2 (2026-07)
